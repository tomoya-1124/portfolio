GitHub Pagesでポートフォリオ公開！詰まったところと学び

投稿日: 2025-09-09 ｜ カテゴリ: Git/GitHub・HTML/CSS/JS

名刺代わりのポートフォリオを GitHub Pages で公開したときの備忘録です。無料で始められて学習にも最適ですが、最初は意外とハマりどころがありました。この記事では、自分が実際に詰まったポイントと解決方法、そして学びをまとめます。

1. 環境と方針

  • 構成：HTML/CSS/JS のみ（学習のため最低限で）
  • 公開：GitHub Pages（リポジトリ名 tomoya-1124.github.io）
  • 目的：名刺代わり＋ブログの土台作り

2. 最初に詰まったところ

2.1 ルートの index.html 問題

トップに表示されるのは リポジトリ直下の index.html。ブログ版を誤って置くと、トップが「Blogだけ」のページになってしまう。

理想の構成:
/
├─ index.html        ← Home（About/Skills/Projects/Contact）
├─ styles.css
├─ script.js
├─ assets/
│   ├─ favicon.ico
│   └─ og-image.png
├─ blog/
│   └─ index.html    ← ブログ一覧
└─ posts/
    └─ first-post.html ← 記事

2.2 Push 時の認証エラー

git push で「Invalid username or token」や「Permission denied (publickey)」に遭遇。HTTPS + PAT か SSH のどちらかで確実に通す必要。

2.3 SSHキーの設定抜け

鍵は作るだけでなく、GitHubのSSH Keysに公開鍵を登録し、ssh-agent に追加するのがポイント。

3. 解決の手順（自分がやったこと）

3.1 フォルダ構成を整える

# クローンして中身を入れ替え
git clone git@github.com:tomoya-1124/tomoya-1124.github.io.git
cd tomoya-1124.github.io
# （ローカルの正しい構成をコピー）
# 例）cp -r ~/se-portfolio-starter/* .
#     cp ~/se-portfolio-starter/.nojekyll .
git add -A
git commit -m "restore structure"
git push origin main

3.2 SSH 認証（通った手順）

# 鍵作成（Enter連打でOK）
ssh-keygen -t ed25519 -C "tom.abe1124@icloud.com"

# エージェント起動＆鍵を登録
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# 公開鍵を表示してGitHubに登録（Settings → SSH and GPG keys）
cat ~/.ssh/id_ed25519.pub

# リモートURLがSSHか確認＆変更
git remote -v
git remote set-url origin git@github.com:tomoya-1124/tomoya-1124.github.io.git

# テスト
ssh -T git@github.com   # Hi tomoya-1124! ... が出ればOK

3.3 GitHub Pages の設定

  • Settings → Pages → Branch: main / フォルダ: /(root)
  • OG画像は絶対URL推奨：og:url と og:image を公開URLで固定

4. 学んだこと

  • トップは必ず /index.html。ブログは /blog/ と /posts/ に分離。
  • Push の詰まりは、認証方式（HTTPS+PAT or SSH）とリモートURLの整合で大体解消。
  • トラブル時は git status と git remote -v、そしてログ/エラーメッセージを素直に読む。

5. 次にやること

  • Contact の文言やSNSリンクを整える
  • favicon と OG画像を自分のブランドに寄せる
  • 技術メモを週1で追加（Knockout.jsのTIPS、境界値テストの具体例など）
