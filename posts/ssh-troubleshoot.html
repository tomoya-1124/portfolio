SSH鍵とpushの詰まりを一気に解消するメモ

投稿日: 2025-09-10 ｜ カテゴリ: Git/GitHub

GitHubで git push が通らない問題を、実体験ベースでまとめました。
「Password authentication is not supported」「Permission denied (publickey)」「Repository not found」あたりを一気に解決します。

1. まずはユーザー情報

コミットできない場合（Author identity unknown）:

git config --global user.name "Tomoya Abe"
git config --global user.email "（GitHubに登録のメール）"

2. SSH鍵を作って登録

ssh-keygen -t ed25519 -C "（メール）"
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
cat ~/.ssh/id_ed25519.pub  # ← これをGitHubの Settings → SSH and GPG keys に貼る

3. リモートURLをSSHに

git remote -v
git remote set-url origin git@github.com:tomoya-1124/tomoya-1124.github.io.git
ssh -T git@github.com   # Hi tomoya-1124! ... が出ればOK

4. それでもダメなら

  • Repository not found → リポジトリ名/ユーザー名のスペル確認
  • Permission denied (publickey) → 鍵の登録漏れ or ssh-add忘れ
  • 詳細は ssh -vT git@github.com のログで確認

5. まとめ

pushの詰まりは、ユーザー情報・SSH鍵・リモートURL の3点チェックで大体解消。
セットアップさえ通れば、以降は git add → commit → push の繰り返しで快適です。
